# Формат коммитов

Проект использует **Conventional Commits** для стандартизации сообщений коммитов.

## Формат коммита

```
type(scope?): subject

body?

footer?
```

### Типы коммитов (type)

- **feat**: новая функциональность
- **fix**: исправление бага
- **docs**: изменения в документации
- **style**: форматирование кода (не влияет на логику)
- **refactor**: рефакторинг кода
- **perf**: улучшение производительности
- **test**: добавление/изменение тестов
- **chore**: рутинные задачи (обновление зависимостей, конфигов)
- **ci**: изменения в CI/CD
- **build**: изменения в системе сборки
- **revert**: откат изменений

### Примеры правильных коммитов

✅ **Хорошие примеры:**

```
feat: добавить виджет расчета ОСАГО
fix(plot): исправить отображение графика на мобильных устройствах
docs: обновить README с инструкциями по установке
refactor(widget): упростить логику расчета стоимости
style: исправить отступы в App.css
test: добавить тесты для компонента Button
chore: обновить зависимости до последних версий
ci: добавить проверку формата коммитов в workflow
build: настроить electron-builder для Windows
```

❌ **Плохие примеры:**

```
Исправил баг
обновление
WIP: новая функция
fix bug
update
```

## Использование

### Способ 1: Интерактивный (рекомендуется)

Используйте Commitizen для создания коммитов в правильном формате:

```bash
npm run commit
```

Это откроет интерактивный интерфейс, где вы выберете тип, scope и опишете изменения.

### Способ 2: Вручную

Просто пишите коммиты в формате Conventional Commits:

```bash
git commit -m "feat(widget): добавить расчет стоимости ОСАГО"
```

### Автоматическая проверка

- **Локально**: При каждом `git commit` автоматически проверяется формат сообщения через Husky hook
- **В CI**: При создании Pull Request проверяются все коммиты

## Установка

После первого `npm install` нужно инициализировать Husky:

```bash
npm run prepare
```

Это автоматически настроит git hooks для проверки коммитов.

## Scope (опционально)

Scope указывает область изменений. Примеры для вашего проекта:
- `widget` - изменения виджета
- `plot` - изменения графика
- `settings` - изменения настроек
- `electron` - изменения Electron приложения
- `ci` - изменения CI/CD

Example: `feat(widget): добавить валидацию полей формы`
